# 系统架构

## 系统上下文

```text
浏览器（React/Vite）
        |
        | HTTPS：静态资源、/api、/uploads
        v
      Nginx
        |
        | HTTP
        v
Go API（Gin：router -> handler -> service -> model）
        |
        | GORM / MySQL 协议
        v
      MySQL 8
```

## 运行时组件

| 组件 | 负责内容 | 不应负责 |
| --- | --- | --- |
| React 前端 | 页面渲染、浏览器交互与状态、API 调用 | 业务持久化规则 |
| Gin 路由与 Handler | 路由、认证中间件、HTTP 参数校验和响应转换 | 核心业务决策 |
| Service 层 | 用例、数据规范化、业务规则和事务边界 | HTTP 或界面细节 |
| Model 层 | 持久化实体和数据表映射 | 请求编排 |
| Database 包 | 数据库连接和连接策略 | 领域行为 |
| Nginx | 静态资源分发和反向代理 | 应用权限认证 |

## 关键流程

读取和写入请求都通过 `/api` 进入系统。路由中间件会在受保护的文档或上传
Handler 执行前校验 `X-Admin-Token`。Handler 将 HTTP 输入转换为 Service
输入；Service 执行业务规则并调用 GORM；Model 定义持久化映射。上传的文件
存储在应用程序二进制之外，并通过 `/uploads` 提供访问。

## 变更规则

- API 契约：说明请求、响应、状态码、兼容性和调用方。
- 数据变更：说明迁移、数据回填、不同版本共存行为和回滚方案。
- 安全变更：说明威胁、信任边界、密钥处理方式和反向测试。
- 依赖变更：说明标准库或现有依赖无法满足需求的原因。
- 跨层功能：按最小垂直切片实现并验证，避免留下互不连通的前后端代码。

架构决策记录在对应的 `specs/changes/*/plan.md` 中。本文描述当前架构；
当已验证的变更修改系统架构时，应同步更新本文。
