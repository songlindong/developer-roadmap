# 项目规格

## 产品定位

Developer Roadmap 是一个私有知识和面试文档站点。读者可以按照分类和标题浏览
Markdown 文档；通过认证的管理员可以创建、编辑、删除文档和上传文档图片。

## 系统约束

- 浏览器端使用 React + Vite，通过 `/api` 与后端通信。
- API 使用 Go + Gin 开发，通过 GORM 访问 MySQL 8。
- 生产环境由 Nginx 提供前端资源，并代理 `/api` 和 `/uploads`。
- 管理员使用 `X-Admin-Token` 认证；密钥通过环境变量提供，禁止提交到仓库。
- 部署必须可以通过 Docker Compose 重现。

## 质量属性

- 每项变更都能从需求追踪到实现和验证证据。
- 除非迁移和回滚方案经过明确批准，否则公开契约和持久化数据必须保持向后兼容。
- 业务规则应能在 Service 层独立测试。
- 前端在桌面端和手机尺寸屏幕上都必须可用。
- API 和基础设施边界发生失败时，应返回安全且可操作的错误信息。

## 完成定义

行为变更只有在满足以下条件后才算完成：所有验收标准和任务已完成、验证证据已记录、
`change.json` 状态已经设为 `verified`，并且 `npm run verify` 执行通过。
